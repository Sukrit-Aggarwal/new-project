Step 3: Component Instances

Description

Component instance is the real deployment of your Component for a given purpose.

A component instance is the way to declare an extremity of a connectivity.

If you want a connection between A and B, A and B have to be declared as component_instance in our AUD world.

The component_instance definition allows you to provide the exact IP (one or many) used for the rules.

The component instance is specific to a landing zone/PaaS. It is not possible to have a component_instance located into 2 landing zones. It has to be split per landing zone
